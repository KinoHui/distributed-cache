项目经验
分布式缓存系统 （2025.02 - 2025.03）
项目架构: 使用了 LRU 缓存淘汰算法、RESTful API、高性能 HTTP 服务端、一致性哈希算法、Singleflight 机制和 Protobuf 技
术栈。
项目描述: 分布式缓存系统是一款高性能的内存数据库，主要包括数据存储、查询、删除等功能，使用了高性能 HTTP 服务端，可以
在高并发场景下有效管理和访问缓存数据，提升数据访问效率，减少后端数据库压力。
项目职责:在本项目中主要负责设计与实现 LRU 缓存模块、RESTful API 接口和一致性哈希算法的优化
1. 实现最近最少使用（LRU）缓存淘汰算法，解决资源限制问题，优化数据访问效率
2. 使用单机并发支持，支持多请求并发处理，提供用户自定义数据源回调函数，提升使用灵活性
3. 实现一致性哈希算法，实现分布式环境中节点选择的一致性哈希，减少节点增减时的数据迁移
4. 使用Singleflight 机制，避免缓存击穿，高并发场景下减轻后端压力，增加后端服务稳定性
